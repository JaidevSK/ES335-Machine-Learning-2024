### Part-1 (Without Momentum)

#### Dataset-1
For Dataset-1, Full Batch Gradient descent converges in around `436` steps/epochs which is faster than stochastic gradient descent which takes around `319245` steps or around `7982` epochs. The noise that is added to the data is very less in magnitude as compared to the magnitude or scale of the data (dataset-1 vary between `-2000` and `2000`). Hence, the ground truth values of the target are more or less linear despite adding the noise. This makes the optimization problem easier and hence the convergence is faster. Stochastic gradient descent takes more time to converge because it updates the weights after every data point (although the stochastic gradient estimator is unbiased it has variance) and hence the updates are very noisy. The noise in the updates makes the optimization problem difficult and hence the convergence is slower.

#### Dataset-2
For Dataset-2, Full Batch Gradient descent converges in around `858` steps/epochs which is faster than stochastic gradient descent which takes around `343223` steps or around `8581` epochs. The noise added added to dataset-2 is more significant as the y values in dataset-2 vary between `-1` and `8`. Hence, the dataset is more noisy, but full batch gradient descent still converges faster than stochastic gradient descent as it is a noisy estimator.

#### Comparing Dataset-1 and Dataset-2
Full Batch Gradient Descent converges faster for both datasets as compared to Stochastic Gradient Descent. The convergence is faster for dataset-1 as compared to dataset-2 as dataset-2 is more noisy. We can notice that Dataset-2 is better suited for stochastic gradient descent as compared to dataset-1. If we don't have enough memory to load the entire dataset into memory, we can use stochastic gradient descent for dataset-2. We observe that if we have to use stochastic gradient descent, it takes around `10` times more epochs to converge as compared to full batch gradient descent for dataset-2 but it takes around `20` times more epochs to converge for dataset-1. Hence, the convergence is faster for dataset-2 as compared to dataset-1. Hence, stochastic gradient descent performs better in the case of dataset-2 than dataset-1 as dataset-2 is more noisy.


### Part-2 (With Momentum)

#### Dataset-1
With momentum, we observe, dataset-1 now takes `344` steps which is faster than `436` steps taken by full batch gradient descent without momentum. This is because momentum helps in reducing the oscillations and hence the convergence is faster, although the cost vs epochs plot is more noiser than without momentum. Stochastic gradient descent takes around `6197361` steps or around `154935` epochs to converge with momentum which is way more than without momentum. This is because the weight updates are still noisy due to the variance of the stochastic gradient estimator. As in stochastic gradient descent, with noisy updates, the direction of each individual gradient step may vary, leading to oscillations and overshooting. Momentum accumulates these inconsistent directions over time, potentially slowing down convergence as it adjusts the current update direction based on historical gradients. When using full batch gradient descent, since we are using the entire dataset for each update we get the true gradient (and not a possibly noisy estimate of a point), and hence the noise from stochastic sampling is eliminated allowing momentum to more efficiently utilize historical gradients and converge in fewer steps than without momentum (for full batch).

#### Dataset-2
Full batch gradient descent takes around `418` steps/epochs to converge with momentum which is faster than `858` steps taken by full batch gradient descent without momentum. Stochastic gradient descent takes around `261833` steps or `6546` epochs to converge with momentum which is faster than `343223` steps or `8581` epochs taken by stochastic gradient descent without momentum. This is because momentum helps in reducing the oscillations and hence the convergence is faster in case of noisier data. The cost vs epochs plot is more noiser than without momentum. As dataset-2 is noisy, the randomness in stochastic gradient descent can potentially help us esacpe local optimas as the parameters are updated more frequently.

General trend is that if data is not noisy, using stochastic gradient descent takes way more steps than full batch gradient descent to converge and with momentum, it takes even more steps, but if the data is noisy, using stochastic gradient descent takes less steps than full batch gradient descent to converge and with momentum, it takes even less steps.
